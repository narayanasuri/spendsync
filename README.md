# expense tracker

A self-hosted personal expense tracker built with Next.js and Supabase. Fork it, configure `config.ts`, point it at your own Supabase project, and you're done.

---

## Stack

- **Next.js 15** (App Router)
- **Supabase** (Postgres + auth via service role key)
- **shadcn/ui** + Tailwind CSS

---

## Quick start

### 1. Fork & clone

```bash
git clone https://github.com/your-username/cheeez
cd cheeez
npm install
```

### 2. Create a Supabase project

1. Go to [supabase.com](https://supabase.com) and create a new project.
2. Once it's ready, open the **SQL Editor** and run the migration below.

### 3. Run the database migration

Paste this into the Supabase SQL Editor and run it:

```sql
create table "Expenses" (
  id          bigint generated by default as identity primary key,
  name        text not null,
  description text default '',
  category    text not null,
  payment_mode text not null,
  amount      numeric not null check (amount > 0),
  spent_at    timestamptz not null default now()
);

-- Enable RLS (the app uses the service role key so all ops are permitted,
-- but it's good practice to have RLS on)
alter table "Expenses" enable row level security;
```

### 4. Set up environment variables

Copy the example file and fill in your values:

```bash
cp .env.local.example .env.local
```

| Variable                    | Where to find it                                                       |
| --------------------------- | ---------------------------------------------------------------------- |
| `SUPABASE_URL`              | Project Settings тЖТ API тЖТ Project URL                                   |
| `SUPABASE_ANON_KEY`         | Project Settings тЖТ API тЖТ `anon` `public` key                           |
| `SUPABASE_SERVICE_ROLE_KEY` | Project Settings тЖТ API тЖТ `service_role` key тАФ keep this secret         |
| `UI_PASSWORD`               | Any password you want тАФ used to protect the UI                         |
| `API_SECRET_KEY`            | Any secret string тАФ used for external API access (e.g. Siri Shortcuts) |

### 5. Customise for your setup

Open **`config.ts`** at the root. This is the only file you need to edit:

- `APP_NAME` тАФ shown in the navbar and login screen
- `CURRENCY` / `LOCALE` / `TIMEZONE` тАФ localisation
- **Categories** тАФ add/remove entries in `CATEGORIES`. Each one needs a `value` (stored in DB), `label` (shown in UI), a [Lucide icon](https://lucide.dev/icons), and an optional `budget`.
- **Payment modes** тАФ add/remove entries in `PAYMENT_MODES`. Drop your bank/wallet logo PNGs into `/public` and reference them via `logoUrl`. Leave `logoUrl` empty to show the fallback initials instead.
- **Budgets / limits** тАФ set `budget` on a category or `limit` on a payment mode to enable the budget consumption and credit usage widgets on the home screen.

### 6. Run locally

```bash
npm run dev
```

## Generating fresh TypeScript types

If you change the database schema, regenerate `lib/database.types.ts` with the Supabase CLI:

```bash
npx supabase gen types typescript --project-id your-project-id > lib/database.types.ts
```

Your project ID is in the Supabase dashboard URL: `https://supabase.com/dashboard/project/<project-id>`.
