# SpendSync - Expense Tracker

A self-hosted personal expense tracker built with Next.js and Supabase. Fork it, point it at your own Supabase project, and you're done.

---

## Stack

- **Next.js 15** (App Router)
- **Supabase** (Postgres + auth via service role key)
- **shadcn/ui** + Tailwind CSS

---

## Quick start

### 1. Fork & clone

```bash
git clone https://github.com/your-username/spendsync
cd spendsync
npm install
```

### 2. Create a Supabase project

1. Go to [supabase.com](https://supabase.com) and create a new project.
2. Once it's ready, open the **SQL Editor** and run the migration below.

### 3. Set up your database schemas

Paste this into the Supabase SQL Editor and run it:

```sql
create table public."PaymentMethods" (
  id bigint generated by default as identity not null,
  name text not null,
  type text not null,
  balance numeric null,
  due smallint not null default '1'::smallint,
  constraint PaymentMethods_pkey primary key (id),
  constraint PaymentMethods_name_key unique (name)
) TABLESPACE pg_default;

create table public."Users" (
  id bigint generated by default as identity not null,
  name text not null,
  "default" boolean null default false,
  constraint Users_pkey primary key (id),
  constraint Users_name_key unique (name)
) TABLESPACE pg_default;

create table public."Categories" (
  id bigint generated by default as identity not null,
  name text not null,
  icon text not null default '❓'::text,
  type text not null default 'expense'::text,
  color text not null default '''#B8D6FB''::text'::text,
  constraint Categories_pkey primary key (id),
  constraint Categories_name_key unique (name)
) TABLESPACE pg_default;

create table public."Expenses" (
  id bigint generated by default as identity not null,
  name text not null,
  description text null default ''::text,
  category bigint not null,
  payment_mode bigint not null,
  spent_at timestamp with time zone not null default now(),
  amount numeric not null,
  transaction_type text not null default '''expense''::text'::text,
  paid_by bigint not null,
  constraint Expenses_pkey primary key (id),
  constraint Expenses_category_fkey foreign KEY (category) references "Categories" (id) on update CASCADE on delete CASCADE,
  constraint Expenses_paid_by_fkey foreign KEY (paid_by) references "Users" (id),
  constraint Expenses_payment_mode_fkey foreign KEY (payment_mode) references "PaymentMethods" (id) on update CASCADE on delete CASCADE,
  constraint Expenses_amount_check check ((amount > (0)::numeric))
) TABLESPACE pg_default;

create table public."Budgets" (
  id bigint generated by default as identity not null,
  category_id bigint not null,
  "user" bigint null,
  budget_amount numeric not null,
  period public.BudgetPeriod not null default 'monthly'::"BudgetPeriod",
  constraint Budgets_pkey primary key (id),
  constraint Budgets_category_id_fkey foreign KEY (category_id) references "Categories" (id),
  constraint Budgets_user_fkey foreign KEY ("user") references "Users" (id)
) TABLESPACE pg_default;

-- Enable RLS (the app uses the service role key so all ops are permitted,
-- but it's good practice to have RLS on)
alter table "PaymentMethods" enable row level security;
alter table "Users" enable row level security;
alter table "Categories" enable row level security;
alter table "Expenses" enable row level security;
alter table "Budgets" enable row level security;
```

### 5. Set up environment variables

Copy the example file and fill in your values:

```bash
cp .env.local.example .env.local
```

| Variable                    | Where to find it                                                       |
| --------------------------- | ---------------------------------------------------------------------- |
| `SUPABASE_URL`              | Project Settings → API → Project URL                                   |
| `SUPABASE_ANON_KEY`         | Project Settings → API → `anon` `public` key                           |
| `SUPABASE_SERVICE_ROLE_KEY` | Project Settings → API → `service_role` key — keep this secret         |
| `UI_PASSWORD`               | Any password you want — used to protect the UI                         |
| `API_SECRET_KEY`            | Any secret string — used for external API access (e.g. Siri Shortcuts) |

### 6. Run locally

```bash
npm i && npm run dev
```

## Generating fresh TypeScript types

If you change the database schema, regenerate `lib/database.types.ts` with the Supabase CLI:

```bash
npx supabase gen types typescript --project-id your-project-id > lib/database.types.ts
```

Your project ID is in the Supabase dashboard URL: `https://supabase.com/dashboard/project/<project-id>`.
